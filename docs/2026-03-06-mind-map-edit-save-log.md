# 구조맵 편집/저장 기능 개발 로그 (2026-03-06)

이 문서는 구조맵 편집/저장 기능을 구현하면서 발생한 문제와 해결 과정을 기록한 개발 로그입니다.

## 1) 기본 테마 저장/적용 경로
- 문제: 기본 맵 테마를 저장할 API가 없고, 본인만 수정 가능하게 보장되어야 함.
- 해결: Next.js API Route에 `/api/profile/theme` 생성 후, 로그인 유저만 `profiles.mind_theme_preference`를 읽고/수정하도록 구현.

## 2) 테마 선택 UI 분리
- 문제: 사용자 메뉴 안의 기본 테마 선택 UI가 비대해져 유지보수가 어려움.
- 해결: 모달 UI를 `MindThemePreferenceModal`로 분리하고 메뉴는 상태/호출만 담당하도록 구조 개선.

## 3) 기본 테마 적용 지연(플래시)
- 문제: 페이지 새로고침 시 기본 테마가 늦게 적용되어 기본 테마 → 사용자 테마로 바뀌는 플래시 발생.
- 해결: 서버 레이아웃에서 `mind_theme_preference`를 읽어 컨텍스트로 전달해 최초 렌더부터 적용.

## 4) 개별 맵 테마 저장/우선 적용
- 문제: 맵별 테마 선택이 DB에 저장되지 않고 전역 테마로만 동작.
- 해결:
  - `maps.mind_theme_override` 저장/조회 API 추가.
  - 맵 로드시 override가 있으면 우선 적용, 없으면 전역 기본 테마 적용.

## 5) 이동 모드에서 노드 선택 가능 문제
- 문제: 이동 모드에서도 노드 선택이 되어 UX 혼란.
- 해결: 이동 모드 시 노드 요소에 `pointer-events: none` 적용하여 선택 차단.

## 6) 보기 모드 컨텍스트 메뉴 편집 제한
- 문제: 보기 모드에서도 우클릭 메뉴에 추가/삭제가 노출됨.
- 해결:
  - 보기 모드에서 편집 관련 작업을 막는 가드 추가.
  - 컨텍스트 메뉴에서 편집 항목 숨김, focus만 유지.

## 7) 자동저장/발행/임시버리기 흐름
- 문제: 자동저장 및 발행/버리기 동작이 없음.
- 해결:
  - Draft API: `/api/maps/[mapId]/draft` (GET/PATCH/DELETE).
  - Publish API: `/api/maps/[mapId]/publish` (POST).
  - 맵 로드시 draft가 있으면 draft 우선 표시.
  - 임시 변경 라벨/버튼 표시 조건 연동.

## 8) 자동저장 실패 및 상태 라벨 표시
- 문제: 자동저장 실패 시 사용자 피드백 부족.
- 해결:
  - 실패 토스트 표시(쿨다운 적용).
  - 저장 중/저장됨/임시변경 있음 상태 라벨 노출.

## 9) 자동저장 중 깜박임/리셋
- 문제: 자동저장 시 맵이 재렌더되어 깜박임 및 뷰 리셋.
- 해결:
  - `onChange` 핸들러를 `ref`로 분리해 재초기화 방지.
  - 재초기화 시 transform/scale 복원하여 뷰 유지.

## 10) Undo/Redo 저장 누락
- 문제: `Cmd+Z` / `Cmd+Shift+Z` 수행 시 변경 감지가 안 되어 draft 저장 누락.
- 해결:
  - `mind.undo` / `mind.redo` 래핑 후 저장 트리거 발생.
  - 저장 대상 op 리스트에 `undo/redo` 포함.

## 11) 보기 모드에서 편집 시도 UX 개선
- 문제: 보기 모드에서 더블클릭 시 사용자 혼란.
- 해결:
  - 더블클릭 시 토스트 안내.
  - 편집 모드 버튼 강조(펄스) 제공.

## 12) 상태 라벨 위치/디자인 조정
- 문제: 라벨이 컨트롤바 안에서 깜박이며 레이아웃이 흔들림.
- 해결:
  - 라벨을 컨트롤바 아래로 이동.
  - 버튼들과 함께 한 줄로 재배치(라벨 → 임시변경 버리기 → 완료/발행).

## 13) 전역 토스트 스타일 통일
- 문제: 일부 토스트만 컬러 적용, 기본 토스트는 흰색.
- 해결: `Toaster`에 전역 스타일을 inline으로 지정해 일관된 컬러 적용.

---

## 현재 구조 요약
- 전역 기본 테마: `profiles.mind_theme_preference`
- 개별 맵 테마: `maps.mind_theme_override`
- 자동저장: `maps.mind_elixir_draft`
- 발행: `mind_elixir_draft → mind_elixir`, draft 삭제
- 로드 우선순위: draft > main
