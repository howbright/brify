# 2026-03-17 Maps Page UX And Preview Devlog

이 문서는 `맵 목록 페이지`, `프리뷰 모드`, `태그 정리 모드`, `상세 페이지 좌측 패널`에 대해 이 스레드에서 논의하고 반영한 내용을 질문-해결 형식으로 정리한 개발 로그다.

## 질문 -> 해결

### 질문
태그 정리 모드에서 프리뷰 켜기 버튼을 눌렀을 때 프리뷰가 보이지 않는다.  
알림만 띄울지, 자동으로 태그 정리 모드를 끄고 프리뷰를 보여줄지 고민이 있다.

### 해결
자동 전환 UX를 채택했다.

- 프리뷰 켜기 시 태그 정리 모드가 켜져 있으면 자동으로 종료
- 토스트로 "프리뷰를 위해 태그 정리 모드를 껐어요." 안내
- 반대로 태그 정리 모드를 켤 때 프리뷰가 켜져 있으면 프리뷰 자동 종료

---

### 질문
프리뷰 모드에서 상단 툴바 버튼 간격이 들쭉날쭉하다.

### 해결
`MapListToolbar`의 버튼 그룹 정렬을 재정비했다.

- 버튼 그룹 사이의 불필요한 시각적 간격 축소
- 데스크톱에서는 한 줄의 일관된 간격으로 정리
- 모바일에서는 별도 줄바꿈 규칙으로 재배치

---

### 질문
`선택`, `태그 정리 모드`, `프리뷰` 버튼이 글자만으로는 직관성이 떨어진다.

### 해결
각 버튼에 의미를 드러내는 아이콘을 추가했다.

- `선택`: 커스텀 체크박스 아이콘
- `태그 정리 모드`: 태그 아이콘
- `프리뷰`: 눈 아이콘

또한 선택 버튼 체크박스 아이콘은 최종적으로 흰색 테두리 + 흰색 체크로 맞췄다.

---

### 질문
필터 버튼을 열었을 때 필터 다이얼로그가 뜨지 않거나, `필터 초기화` 버튼을 한 번 눌러서는 초기화되지 않는 문제가 있다.

### 해결
원인은 오버레이와 툴바 버튼의 레이어 충돌이었다.

- 필터 팝오버 오버레이가 툴바를 덮지 않도록 구조 조정
- 필터 버튼/초기화 버튼이 오버레이보다 위에 오지 않도록 모바일 z-index 정리
- 이후 `필터 열기 -> 초기화`가 바로 동작하도록 수정

---

### 질문
태그 정리 모드에서 맵 제목 검색과 좌측 태그 필터가 동시에 적용되면 헷갈린다.

### 해결
태그 정리 모드의 필터 책임을 단순화했다.

- 태그 정리 모드에서는 제목 검색창 숨김
- 태그 정리 모드에서는 필터 다이얼로그의 태그 필터 숨김
- 태그 관련 필터는 좌측 태그 패널만 사용

---

### 질문
태그 정리 모드에서 테이블 태그 컬럼이 들쭉날쭉하고, 태그가 1개여도 제대로 보이지 않는다.

### 해결
태그 컬럼을 `2줄 + N` 방식으로 정리했다.

- 태그 영역 최소 높이 고정
- 최대 2줄 표시
- 초과 태그는 `+N` 표시
- 태그 없을 때는 `-` 표시
- 편집 버튼 위치 고정

---

### 질문
태그 정리 모드에서 상세 페이지로 이동할 방법이 없다.

### 해결
`열기` 버튼을 추가했다.

- 카드뷰와 테이블뷰 모두 지원
- 이후 일반 모드에서도 `열기` 버튼을 유지
- 행 클릭은 상세 이동이 아니라 선택/프리뷰 동작만 하도록 변경
- 상세 이동은 `열기` 버튼으로만 수행

---

### 질문
`열기` 버튼이 제목 길이에 따라 위치가 흔들린다.

### 해결
제목 컬럼과 버튼의 레이아웃을 고정했다.

- 제목은 `truncate`
- `열기` 버튼은 `shrink-0`
- 카드뷰에서는 `열기` 버튼을 우측 상단으로 이동

또한 `열기`, `편집` 버튼 모두 hover와 cursor 스타일을 명확하게 했다.

---

### 질문
제목 정렬이 한국어 사용자 기대와 다르게 보인다.

### 해결
우선 DB 변경 없이 `Intl.Collator(locale)` 기반 프론트 정렬을 적용했다.

- 제목 정렬 시 현재 locale 기준 문자열 비교
- 현재는 임시 해결책이며, 다국어 + 페이지네이션 정합성까지 완벽히 맞추려면 향후 DB 정렬(RPC/ICU collation) 검토 필요

---

### 질문
태그 편집 후 좌측 태그 목록 숫자가 바로 갱신되지 않는다.

### 해결
태그 편집 저장 후 `tagRefreshKey`를 증가시켜 태그 목록을 다시 fetch하도록 수정했다.

---

### 질문
태그 정리 모드의 태그 선택 UX가 혼동된다.  
행 클릭 필터와 체크박스 선택이 별도로 존재한다.

### 해결
체크박스 기반 다중 OR 필터로 통합했다.

- 태그 정리 모드에서는 체크박스 선택 상태 자체가 필터 상태
- 여러 태그를 동시에 선택 가능
- 조건은 OR
- 상단에는 선택된 태그를 검은 칩으로 표시
- 칩의 `X`로 개별 해제 가능

추가로:

- `필터링 중` 라벨을 칩 위 제목처럼 배치
- `태그합치기` 버튼은 항상 보이되, 실제 태그 2개 이상 선택 시에만 활성화
- `특별한 태그없음`은 필터용 가상 항목으로 추가
- `특별한 태그없음`은 병합 다이얼로그/병합 대상에서 제외

---

### 질문
태그 목록이 `최근 태그`, `전체 태그`로 나뉘어 있는데 오히려 복잡하다.

### 해결
하나의 통합 리스트로 단순화했다.

- 정렬 기준 4개 지원
- 기본값은 `최신`
- 지원 정렬:
  - 최신
  - 이름순
  - 많이 쓰인 순
  - 적게 쓰인 순

---

### 질문
모바일에서 태그 정리 모드를 켜면 태그 패널이 아예 보이지 않는다.

### 해결
모바일용 태그 정리 바텀시트를 추가했다.

- 태그 정리 모드 진입 시 모바일에서는 바텀시트로 태그 패널 제공
- 시트를 닫아도 태그 정리 모드는 유지
- 하단에 `태그 정리 다시 열기` 버튼 제공
- 오버레이/z-index 충돌도 함께 수정

---

### 질문
태그 바텀시트에서 태그를 클릭할 때마다 태그 리스트가 깜박인다.

### 해결
태그 패널을 전역 목록 로딩 상태와 분리했다.

- 맵 리스트 로딩과 태그 리스트 로딩을 분리
- 태그 패널은 `tagsLoading`일 때만 내부 스켈레톤 표시
- 태그 선택 시 맵 리스트만 재로딩

---

### 질문
프리뷰 로드 시 맵이 중앙에 갔다가 다시 다른 자리로 움직이거나, 맵을 여러 개 바꿔 클릭하면 엉뚱한 위치가 나온다.

### 해결
미리보기의 view state 복원 로직을 분리했다.

- 상위 프리뷰 패널의 중복 centering 제거
- `ClientMindElixir`에 `preserveViewState` 옵션 추가
- `MapMiniPreview`에서는 `preserveViewState={false}` 적용
- 이전 맵의 pan/zoom 상태가 다음 미리보기에 섞이지 않게 수정

---

### 질문
맵 리스트 화면 로딩 시 텍스트만 보여서 실서버에서는 체감이 나쁠 수 있다.

### 해결
스켈레톤 UI를 추가했다.

- 카드뷰 스켈레톤
- 테이블뷰 스켈레톤
- 태그 패널 스켈레톤
- 프리뷰 패널 스켈레톤

---

### 질문
맵 리스트 상단 검색/버튼 영역이 스크롤 시 사라져서 불편하다.

### 해결
데스크톱에서만 sticky 툴바로 변경했다.

- 데스크톱: sticky
- 모바일: 일반 흐름 유지
- 이후 모바일 레이아웃도 별도로 재정비

---

### 질문
상세 페이지 닫기 버튼에서 locale 없는 경로로 이동하면서 에러가 발생한다.

### 해결
닫기 이동 경로를 locale-aware 하게 수정했다.

- 기존: `/maps`
- 변경: `/${locale}/maps`

---

### 질문
상세 페이지 좌측 패널의 `노트`, `용어` 탭에 현재 개수를 표시하고 싶다.

### 해결
탭 라벨 옆에 숫자 배지를 추가했다.

- 노트 개수 배지
- 용어 개수 배지
- `0`일 때는 숨김
- 비활성 탭일 때도 푸른 계열 톤 유지

또한 탭을 누르지 않아도 배지가 보이도록 패널이 열릴 때 백그라운드 fetch를 추가했다.

---

### 질문
노트 입력창에서 Enter를 누르면 같은 노트가 두 번 추가된다.

### 해결
중복 제출 방어를 추가했다.

- `noteSubmitting` 상태 추가
- 요청 중 Enter 재입력 무시
- 요청 중 버튼 비활성화

---

### 질문
`DiscardDraftDialog`의 상단 닫기 버튼이 깨지고, 문구가 과하다.

### 해결
상단 닫기 버튼을 텍스트 없는 `X` 아이콘만 남기는 형태로 단순화했다.

---

### 질문
`maps/page.tsx`의 다크모드에서 페이지 전체 배경이 흰색으로 보인다.

### 해결
페이지 루트에 다크 배경을 추가했다.

- 라이트: `bg-neutral-50`
- 다크: `bg-[#07111f]`

---

### 질문
프리뷰 패널에는 `열기`만 있고 `닫기`가 없다.

### 해결
프리뷰 패널 헤더에 `닫기` 버튼을 추가했다.

- `열기`: 상세 페이지 이동
- `닫기`: 프리뷰 모드 종료

## 메모

- 제목 정렬의 완전한 다국어 정합성은 장기적으로 DB 레벨 정렬(RPC + ICU collation) 검토 필요
- 현재는 프론트 `Intl.Collator` 정렬로 우선 대응
- 태그 정리 모드의 모바일 UX는 바텀시트 방식으로 1차 지원 완료
