# 구조맵 생성 흐름 Q&A 로그 (열기 버튼 → FullscreenDialog)

범위: `DraftMapCard.tsx`의 "열기" 버튼으로 시작해 `FullscreenDialog.tsx` 내부 기획/구현 진행까지의 흐름.
형식: 질문 -> 문제해결

---

## 질문 -> 문제해결

1) 질문: `DraftMapCard.tsx`의 "열기" 버튼을 누르면 어떤 화면이 열리며, 구조맵 화면을 어떻게 설계할까?  
   문제해결: `FullscreenDialog.tsx`를 전체 화면 컨테이너로 사용하고, 좌/우 패널과 중앙 맵 영역으로 UI 골격을 확정했다. 내부 컨텐츠를 확장 가능한 구조로 설계해 이후 메타/상세 정보를 자연스럽게 붙일 수 있도록 했다.

2) 질문: "열기" 버튼은 언제 클릭 가능해야 할까?  
   문제해결: 상태가 `processing`이면 "열기" 버튼을 비활성화하고, 대신 "정보 수정"은 항상 열 수 있게 분리했다. 사용자 흐름을 끊지 않으면서, 처리 중인 맵은 열리지 않도록 보호했다.

3) 질문: 처리 중 상태를 사용자가 지루하지 않게 이해하도록 표현할 수 있을까?  
   문제해결: `DraftMapCard.tsx`의 `processing` 라벨을 단일 텍스트에서 순환 메시지로 변경했다. 한국어 문구가 일정 간격으로 교체되도록 설계해 기다림을 덜 답답하게 만들었다.

4) 질문: 메타데이터 편집은 생성 전/후 모두 필요하지 않을까?  
   문제해결: `MetadataDialog.tsx`를 `mapId` 기반으로 열 수 있게 수정하여, 생성 전/후 모두 동일한 UI에서 정보를 수정할 수 있도록 했다.

5) 질문: 썸네일 업데이트가 적용되지 않는 이유는 무엇일까?  
   문제해결: 브라우저 캐시 이슈로 판단하고, `thumbnail_url`에 타임스탬프를 붙이는 캐시 버스팅을 적용해 즉시 업데이트되도록 해결했다.

6) 질문: `FullscreenDialog.tsx`가 화면을 완전히 덮지 못하는 문제는 왜 생겼을까?  
   문제해결: 내부 컨테이너의 `h-full w-full`, `p-0` 등을 정리하고 `open` prop 처리를 복원해 전체 화면과 상태 표시가 안정적으로 동작하도록 수정했다.

7) 질문: 좌측 패널 정보 UI는 어디에 두는 게 유지보수에 좋을까?  
   문제해결: `FullscreenDialog.tsx`에서 `LeftPanel.tsx`로 분리했다. `MapMeta` 타입과 `SidePanel/MetaBlock/InfoItem` 컴포넌트를 함께 이동해 재사용과 관리 범위를 명확히 했다.

8) 질문: 우측 패널의 "노트/용어"는 실제 제품 요구와 맞을까?  
   문제해결: 기존 탭 구조를 제거하고 DB `maps` 테이블 기준으로 "상태/콘텐츠/소스/시스템" 섹션으로 재설계했다. `RightPanel.tsx`에 `MapRow` 기반의 명확한 데이터 표현을 구축했다.

9) 질문: 날짜가 SSR/CSR에서 다르게 렌더링되어 Hydration 에러가 나는 문제는?  
   문제해결: `RightPanel.tsx`에서 날짜 포맷을 미리 문자열로 만들도록 변경해 서버/클라이언트 불일치를 제거했다.

10) 질문: 마인드맵 스타일이 확실히 적용되지 않는 이유는?  
    문제해결: `ClientMindElixir.tsx`와 `globals.css`에서 MindElixir 내부 요소에 `!important`를 포함한 명시적 스타일을 적용하고, 배경 요소는 `transparent`로 강제했다.

11) 질문: 노드 접기/펼치기 시 스타일이 사라지는 문제는?  
    문제해결: `MutationObserver`로 DOM 변화를 감지해 스타일 재적용을 수행했다.

12) 질문: 노드 편집 중 텍스트가 겹치거나 사라지는 문제는?  
    문제해결: `beginEdit`에서 겹치는 텍스트를 숨기고, `finishEdit/cancelEdit/endEdit`에서 복원하도록 처리했다. 이벤트 누락 상황을 대비해 편집 상태가 없으면 자동 복원하는 안전장치도 추가했다.

13) 질문: 우측 패널과 좌측 패널이 동시에 열려 UI가 답답해지는 문제는?  
    문제해결: `openMeta` 동작에서 반대쪽 패널을 자동으로 닫도록 수정해 한 번에 하나의 패널만 열리도록 정리했다.

14) 질문: "페이지로 열기" 링크가 사라진 이유는?  
    문제해결: 좌측 패널 분리 과정에서 누락된 링크를 `LeftPanel.tsx`에 복구했다.
