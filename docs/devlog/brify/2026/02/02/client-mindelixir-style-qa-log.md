# ClientMindElixir 스타일 Q&A 로그

범위: `ClientMindElixir.tsx` 및 관련 스타일(`globals.css`)에서 발생한 UI/스타일 문제와 해결 과정.
형식: 질문 -> 문제해결

---

## 질문 -> 문제해결

1) 질문: 마인드맵의 배경 스타일이 적용되지 않고 흰색으로 보이는 이유는?  
   문제해결: MindElixir 내부 요소들이 기본 흰색 배경을 갖고 있어, `.map-container`, `.map-canvas`, `me-nodes` 등 여러 내부 요소에 `background: transparent !important`를 지정해 커스텀 배경이 드러나도록 했다.

2) 질문: 루트 노드/리프 노드 스타일이 적용되지 않거나 기본 테두리가 남는 이유는?  
   문제해결: MindElixir 기본 스타일이 우선되어 있었기 때문에, `globals.css`에서 선택자를 강화하고 `!important`를 추가해 커스텀 배경/텍스트/테두리 규칙이 확실히 적용되도록 정리했다.

3) 질문: 리프 노드가 접힘/펼침 후 스타일이 사라지는 문제는?  
   문제해결: DOM 재구성이 발생할 때 스타일이 초기화되는 것을 확인하고, `MutationObserver`로 변화 감지 후 스타일 재적용을 수행했다.

4) 질문: 노드 편집 시 텍스트가 겹치거나 사라지는 문제는?  
   문제해결: `beginEdit` 시점에 겹치는 텍스트를 숨기고, `finishEdit/cancelEdit/endEdit`에서 복원하도록 처리했다. 이벤트 누락에 대비해 편집 상태가 없으면 자동 복원하는 안전장치도 추가했다.

5) 질문: 노드 카드 외곽선이 두 겹으로 보이는 문제는?  
   문제해결: 내부 텍스트 영역에 중복 테두리가 적용되는 것을 제거하고, 카드 단위의 테두리 규칙만 남겨 시각적 중복을 해결했다.

6) 질문: 맵 영역 주변 여백이 과하게 보이는 이유는?  
   문제해결: `ClientMindElixir.tsx`의 래퍼 `div` 패딩을 `p-3`에서 `p-0`으로 변경해 불필요한 여백을 제거했다.

7) 질문: 스타일 변경이 적용되었는데도 일부 상태에서 다시 원복되는 이유는?  
   문제해결: MindElixir의 내부 렌더 사이클이 스타일을 덮어쓰는 경우가 있어, 스타일 적용 함수를 재호출하는 구조로 안정화했다.
