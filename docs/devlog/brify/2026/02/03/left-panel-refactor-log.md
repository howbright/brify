# Left Panel UI 리팩토링 작업 로그

## 파일명
left-panel-refactor-log.md


## 1. 질문 → 문제해결  
**LeftPanel에서 description이 제목 바로 아래에 있는데, 내용이 길어질 수 있어서 부담스럽다. 위치를 바꾸고 싶다.**

### 문제 인식
- description이 제목 바로 아래에 노출되면서 헤더 영역이 과도하게 길어짐
- 패널을 열었을 때 핵심 정보(제목, 출처)가 바로 보이지 않음
- description은 길이가 예측 불가능한 텍스트라 항상 노출되는 구조에 부적합

### 해결
- description을 헤더 영역에서 제거
- body 영역에 별도의 `설명` 섹션으로 분리
- 기본 상태는 접힘, 버튼 클릭 시 펼쳐지는 구조로 변경
- 긴 텍스트여도 UI가 밀리지 않도록 안정적인 레이아웃 유지


## 2. 질문 → 문제해결  
**created_at / updated_at 시간이 너무 강조되어 보인다. 존재감을 확 줄이고 싶다.**

### 문제 인식
- 시간 정보가 Section + RowItem 구조로 들어가 시각적 비중이 과도함
- 사용자가 중요 정보처럼 인식하게 되어 읽기 흐름을 방해
- 이 패널의 목적은 메타데이터 관리가 아니라 콘텐츠 탐색임

### 해결
- 시간 전용 Section 제거
- 헤더 하단에 작은 텍스트 한 줄로 통합
- font-size를 줄이고 색상을 흐리게 처리하여 존재감 최소화
- 정보는 유지하되 시각적 부담을 크게 감소


## 3. 질문 → 문제해결  
**크레딧 정보도 너무 눈에 띈다. 소요된 크레딧만 작게 한 줄로 보여주고 싶다.  
required_credits, charged_at은 이 컴포넌트에서 제거하고 싶다.**

### 문제 인식
- 카드 형태의 크레딧 UI가 실제 중요도 대비 과도하게 강조됨
- 관리 화면처럼 보이는 인상을 줌
- 사용자가 반드시 알아야 할 정보는 `실제로 소요된 크레딧`뿐임

### 해결
- 크레딧 Section 전체 제거
- 헤더 하단에 `소요 크레딧: {credits_charged}` 한 줄만 표시
- required_credits, credits_charged_at 관련 렌더링 코드 제거
- UI 밀도와 시선 분산 문제 해결


## 4. 질문 → 문제해결  
**상태(Status) 섹션이 필요 없다. 제거하고 싶다.**

### 문제 인식
- extract 상태, schema_version, output_language 등은 개발/디버깅용 정보
- 일반 사용자에게는 정보 과잉
- 패널이 점점 어드민/디버그 UI처럼 보이기 시작함

### 해결
- 상태(Status) Section 전체 제거
- 관련 Badge, KeyValue 렌더링 제거
- 사용자 중심의 단순한 정보 구조로 정리


## 5. 질문 → 문제해결  
**데이터(Data) 섹션도 필요 없다. mind_elixir, extracted_text 등은 빼고 싶다.**

### 문제 인식
- JSON 요약, 텍스트 미리보기는 디버깅에는 유용하나 사용자 UI에는 불필요
- 패널의 복잡도를 크게 증가시킴
- 핵심 정보 파악을 방해함

### 해결
- 데이터(Data) Section 전체 제거
- summarizeJson, summarizeText 사용처 제거
- LeftPanel을 사용자 친화적인 콘텐츠 상세 패널로 정리


## 최종 정리
- LeftPanel의 역할을  
  디버그/메타데이터 패널 → 콘텐츠 중심 상세 패널로 재정의
- 핵심 정보만 빠르게 파악 가능하도록 구성
  - 제목
  - 출처
  - 태그
- 부가 정보는
  - 작게
  - 한 줄로
  - 필요할 때만 확인 가능하도록 정리
