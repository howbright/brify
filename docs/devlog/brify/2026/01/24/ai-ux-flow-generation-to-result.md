# AI 구조맵 생성 서비스 – 실무 UX 설계 & 구현 기록

이 문서는  
**신입 기획자 / 개발자 / 1인 창업자**를 위해  
AI 기반 서비스에서 *“입력 → 처리 → 결과”* 플로우를  
실무적으로 어떻게 설계하고 구현했는지를 정리한 기록입니다.

단순 기능 구현이 아니라,  
**사용자가 실제로 ‘좋다’고 느끼는 UX를 만드는 과정**에 초점을 둡니다.

---

## 1. 문제 정의: “AI가 일은 했는데, 결과가 안 느껴진다”

초기 구현 상태에서는 다음과 같은 문제가 있었습니다.

- 사용자가 `생성하기` 버튼을 누름
- AI 처리가 끝남
- 입력창이 다시 활성화됨
- 하단에 Draft 카드가 하나 추가됨

### 문제점
- **“방금 만든 결과”가 메인에 보이지 않음**
- 사용자는 *내가 뭘 만들었는지* 즉시 체감하지 못함
- 결과보다 다시 입력이 강조되는 느낌

👉 AI 서비스에서는  
**“처리 완료 순간의 UX”가 제품 신뢰도를 크게 좌우**합니다.

---

## 2. 핵심 결정: 완료 즉시 ‘결과 패널’을 메인으로 보여준다

여러 선택지 중 최종적으로 선택한 방향은 다음입니다.

### 선택한 UX
> **AI 처리 완료 즉시, 입력 영역을 결과 구조화 패널로 교체**

- 입력 textarea는 숨김
- “생성 완료되었습니다” 메시지를 메인에 표시
- 다음 행동을 명확히 제시

### 결과 패널의 역할
- 지금 무슨 일이 끝났는지 명확히 전달
- 사용자가 다음에 무엇을 하면 되는지 안내

---

## 3. ResultReadyPanel 설계 (결과 완료 패널)

결과 패널에는 **딱 두 가지 행동만 제공**합니다.

### Primary Action
- **구조맵 열기**
- `/maps/[id]` 상세 페이지로 이동

### Secondary Action
- **다른 콘텐츠 생성하기**
- 입력 화면으로 돌아가기 (자동 복귀 ❌)

### 왜 자동 복귀를 안 했는가?
- 사용자가 결과를 충분히 인식하기 전에 UI가 바뀌면 혼란
- “다시 만들기”는 **사용자의 명시적 선택**이어야 더 제품스럽다

---

## 4. Metadata 입력 단계에서의 중요한 UX 판단

### 문제
- `나중에 입력하기` 버튼이 존재
- 메타데이터 없이도 작업이 진행될 수 있는 구조

### 판단
> **제목은 반드시 필요하다**

이유:
- 목록, 결과 카드, 상세 페이지의 기준점
- 나중에 관리/검색/공유에 모두 영향

### 최종 결정
- `나중에 입력하기` 버튼 제거
- 제목 미입력 시:
  - 저장 불가
  - 닫기(X) / backdrop 클릭도 차단
- 나머지 필드(채널, 태그, 설명 등)는 선택

👉 **“최소 필수만 강제”**하는 전략

---

## 5. AI 처리 중 UI에서 신경 쓴 부분

### 문제
- 다크모드에서 진행 상태가 잘 안 보임
- 배경과 진행바가 너무 비슷한 톤

### 개선
- 상단에 compact한 AI 상태 바 배치
- 진행바 색상을:
  - 블루 → 그린 → 퍼플 그라데이션
- 다른 UI와 확실히 다른 시각적 포인트 부여

👉 AI가 “일하고 있다”는 느낌을 명확히 전달

---

## 6. 전체 플로우 구조화

1. 사용자가 콘텐츠를 붙여넣음
2. 언어 선택 후 `생성하기`
3. 크레딧 확인
4. Metadata 입력 (제목 필수)
5. AI 처리 진행
6. 완료 즉시 ResultReadyPanel 표시
7. 사용자가 선택
   - 구조맵 열기 → 상세 페이지
   - 다른 콘텐츠 생성하기 → 입력 화면

---

## 7. 이 영상/기록이 도움이 되는 대상

### 신입 기획자
- 기능이 아니라 **플로우 단위로 UX를 설계하는 방법**
- “왜 이 버튼은 없어야 하는가”에 대한 사고 방식

### 신입 개발자
- 단순 구현이 아닌 **상태 전환 기반 UI 설계**
- AI 비동기 작업을 UI에 자연스럽게 녹이는 방식

### 1인 창업자
- MVP 단계에서 어디까지를 강제하고,
  어디를 선택으로 남겨야 하는지에 대한 판단 기준
- “완성도는 디테일에서 나온다”는 실제 사례

---

## 마무리

AI 서비스를 만든다는 것은  
**AI를 잘 쓰는 것보다, 결과를 잘 ‘보여주는 것’이 더 중요**합니다.

이 기록이  
“기능은 되는데 뭔가 어색한 서비스”를  
“쓰고 싶은 제품”으로 바꾸는 데 도움이 되길 바랍니다.
