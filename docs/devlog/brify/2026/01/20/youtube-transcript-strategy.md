# Brify 개발 로그 #1  
## 유튜브 URL로 스크립트 가져오기 – 기술 전략 세우기

이 영상은 **실제 SaaS 서비스를 개발하면서**
AI(ChatGPT)와 함께 **기술적인 의사결정을 어떻게 하는지**를
그대로 녹화한 개발 로그입니다.

단순한 튜토리얼이 아니라,
> ❝ 왜 이 구조를 선택했는지 ❞  
> ❝ 어떤 선택지를 고민했고, 무엇을 버렸는지 ❞  
를 중심으로 이야기합니다.

---

## 🎯 이번 영상에서 다룬 내용

- 유튜브 URL로 스크립트를 가져오는 기능을 어떻게 설계할까?
- Next.js API vs 별도 Backend(NestJS) 중 어디에서 처리하는 게 맞을까?
- 응답 시간이 길어질 가능성은?
- 3시간짜리 영상처럼 **스크립트가 매우 긴 경우**는 어떻게 안전하게 처리해야 할까?
- 큐(BullMQ, Worker)는 처음부터 필요할까, 아니면 나중에 붙여도 될까?
- “지금 단계에서” 가장 현실적인 기술 선택은 무엇일까?

---

## 🧠 이 영상이 도움이 되는 사람

- Next.js / NestJS로 실제 서비스를 만들고 있는 개발자
- AI를 “코드 생성기”가 아니라 **사고 파트너**로 쓰고 싶은 사람
- 서버리스 vs 백엔드 분리 구조를 고민 중인 사람
- 대용량 텍스트 / 장시간 콘텐츠 처리 전략이 궁금한 사람
- 포트폴리오가 아니라 **진짜 서비스 개발 과정**이 보고 싶은 사람

---

## 🧩 이 시리즈의 방향

이 영상은 “정답을 알려주는 튜토리얼”이 아니라,
**현업 개발자가 실제로 고민하고 결정하는 과정을 기록**합니다.

- (1) 기술 전략 세우기 ← 이번 영상
- (2) NestJS에 스크립트 추출 API 구현
- (3) 긴 스크립트 안전하게 저장/처리하기
- (4) 구조화 & 구조맵 생성으로 확장

👉 실패한 선택, 바뀐 결정도 그대로 남깁니다.
