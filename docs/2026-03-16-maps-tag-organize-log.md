# 2026-03-16 Maps Tag Organize & List UX 개발로그

## 문제 -> 해결
- 리스트가 많아질 때 전체 펼침으로 성능/UX 저하 -> 서버 페이지네이션 및 필터 기반 로딩 구조로 정리.
- 필터 영역이 페이지 공간을 과하게 차지함 -> 다이얼로그(팝오버) 형태로 변경하고 스크롤 처리.
- 필터 UI 간격이 `space-y` 미적용 -> `flex flex-col gap-*`로 전환.
- 태그 목록이 너무 흐릿하고 가독성 낮음 -> 프로젝트 톤에 맞춘 진한 회색/블루 계열로 강조.
- 태그 리스트가 많을 때 스크롤 불가 -> 최대 높이와 스크롤 적용.
- 태그 정리 모드에서 상세 이동이 우선되어 드래그/정리 어려움 -> 클릭 동작 제어 및 모드별 동작 분리.
- 태그 목록에서 인기/최근/전체 구성 혼란 -> 최근/전체만 유지, 전체는 인기순 정렬.
- 태그 필터가 로컬 데이터만 필터링해 결과가 비어 보임 -> 서버 쿼리에 태그 필터를 직접 반영.
- 태그 편집 UX가 부자연스러움 -> 칩(Chip) 기반 편집 UI로 변경.
- 한글 입력 시 태그가 쪼개지는 이슈 -> IME 조합(composition) 상태 처리로 해결.
- 태그 편집 버튼이 항상 보여 시선 분산 -> 태그 정리 모드에서만 노출.
- 태그 편집 다이얼로그 내 안내/레이블 톤 개선 -> 라벨 컬러 조정 및 불필요 문구 제거.
- 태그 편집 다이얼로그 재사용성 부족 -> 컴포넌트 분리.
- 태그 추가/삭제/합치기 API 누락 -> `/api/maps/tags/add`, `/api/maps/tags/remove`, `/api/maps/tags/update`, `/api/maps/tags/merge` 추가.
- 태그 합치기 흐름 부재 -> 태그 체크 선택 + “태그합치기” 액션 + 기준 태그 선택 다이얼로그로 구현.
- 합치기 기준 태그가 외부 태그 제안으로 혼란 -> 선택한 태그만 기준 후보로 노출.
- 태그 정리 패널의 수동 태그 추가가 불필요해짐 -> 태그 리스트 필터(검색)로 전환, X 초기화 추가.

## 문제 -> 해결 (UI 구조 정리)
- 페이지 코드가 과도하게 커짐 -> 주요 영역 컴포넌트 분리:
  - `TagPanel`
  - `MapListToolbar`
  - `MapFilterPopover`
  - `MapCardList`
  - `MapTableList`
  - `SelectionActionBar`
  - `TagEditDialog`
  - `TagMergeDialog`

## 문제 -> 해결 (리스트/테이블 UX)
- 프리뷰/선택 모드 충돌 -> 모드 전환 시 상호 비활성화 및 안내 토스트.
- 테이블 컬럼 비율 불균형 -> 컬럼 고정폭/가변폭 재조정.
- 페이지네이션 가시성 부족 -> 고급스러운 짙은 회색 톤으로 개선.
- 검색창/필터/정렬 UI 대비 부족 -> 진한 테두리 및 톤 통일.

## 산출물
- 태그 정리 모드: 검색 필터, 선택 합치기, 태그 삭제, 태그 편집(칩)
- 필터/정렬/검색/뷰 전환을 모듈화한 Toolbar
- 카드/테이블 리스트 분리 및 유지보수성 개선
- 태그 관련 서버 API 확장
