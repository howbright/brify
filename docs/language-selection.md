# 구조맵 출력 언어 선택 기능

이 문서는 구조맵 생성 시 출력 언어를 선택할 수 있도록
기능을 설계하고 구현한 과정을 정리한 기록입니다.

---

## 1. 왜 이 기능이 필요했는가

- 서비스 UI는 한국어/영어만 지원
- 하지만 결과물(구조맵)은 전 세계 언어로 필요
- 입력 텍스트와 출력 언어가 다를 수 있음

그래서 **출력 언어를 명시적으로 선택하는 기능**이 필요했습니다.

📺 관련 영상: 00:02:10 ~ 00:04:30

---

## 2. 처음 떠올린 선택지들

### 선택지 A
- 결과 생성 후 번역 API를 한 번 더 호출

### 선택지 B
- 생성 전에 출력 언어를 선택하고
- 처음부터 해당 언어로 구조맵 생성

📺 관련 영상: 00:05:00 ~ 00:07:40

---

## 3. 최종 결정과 이유

**선택: B (생성 전에 출력 언어 선택)**

이유:
- 비용 예측이 쉬움
- UX가 자연스럽다
- 결과 품질이 더 좋다

📺 결정 시점: 00:08:10

---

## 4. 구현 중 실제로 마주친 문제

### 문제
- 더보기 모달에서 선택한 언어가
- 드롭다운(퀵 옵션)에 표시되지 않음

📺 문제 발견: 00:14:20

---

## 5. 문제의 원인

HTML `<select>`는  
`option`에 없는 value를 표시할 수 없다.

즉:
- quick 옵션에 없는 언어를 선택하면
- select에서 보여줄 수 없었다.

---

## 6. 해결 방법

- 현재 선택된 언어가 quick 옵션에 없을 경우
- 임시 option을 주입하여 표시

```ts
if (!quickOptions.includes(value)) {
  selectOptions = [
    { code: value, label: `현재 선택: ${label}` },
    ...quickOptions,
  ];
}
