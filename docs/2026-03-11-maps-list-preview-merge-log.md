# 2026-03-11 맵 목록/프리뷰/합치기 개발 로그

아래는 이번 작업에서 발생한 문제와 해결 내용을 "문제 -> 해결" 형식으로 정리한 개발 로그입니다.

## 1) 미니 프리뷰 초기 깜박임
- 문제 -> 프리뷰 로딩 과정에서 "준비중" → 샘플 맵 → 실제 프리뷰 순으로 화면이 여러 번 바뀌며 깜박임 발생.
- 해결 -> `ClientMindElixir`에 `allowSampled` 옵션을 추가하여 미니 프리뷰에서는 샘플 맵을 사용하지 않도록 함. `MapMiniPreview`에서 `allowSampled={false}` 적용.

## 2) 미니 프리뷰 컨트롤 개선
- 문제 -> 우측 하단 줌 버튼만 존재, 센터 맞춤 버튼이 필요함.
- 해결 -> 센터 버튼 추가 및 위치 조정. 아이콘을 더 분명하게 변경.

## 3) 프리뷰 패널 버튼/문구 정리
- 문제 -> "미리보기 닫기" 버튼 제거 필요, "Open Map" 한글화 요청.
- 해결 -> 닫기 버튼 제거, "Open Map"을 "열기"로 변경.

## 4) 목록 대량 대응 (페이지네이션)
- 문제 -> 전체 목록을 한 번에 불러오면 1,000~10,000개 규모에서 성능 저하.
- 해결 -> 서버 페이지네이션 적용 (`range(from, to)`), 하단 페이지네비게이션 추가 및 sticky 적용.

## 5) 검색 필터링 범위 개선
- 문제 -> 태그/설명 검색 필요, 태그 부분매칭 쿼리 오류 발생.
- 해결 -> 태그는 `tags.cs` 정확 매칭으로 유지, 제목/설명은 부분 매칭(`ilike`) 적용. 태그 부분매칭은 DB 확장 전까지 보류.

## 6) 필터 패널 UX 개선
- 문제 -> 필터 영역이 인라인 확장되어 리스트 레이아웃이 밀림.
- 해결 -> 필터 팝오버 형태로 변경 (데스크탑), 인라인 버튼에서 열고 닫도록 수정.

## 7) 필터 기능 확장
- 문제 -> 상태/소스/태그/기간 필터가 필요함, 기본은 최근 7일.
- 해결 -> 프리셋 기간(7/30/90/1년/전체) + 커스텀 기간 추가. 그룹 내 OR, 그룹 간 AND 규칙 적용. 기본은 최근 7일.

## 8) 태그 필터 서버 집계
- 문제 -> 태그 목록을 클라이언트에서 만들면 성능 저하.
- 해결 -> `/api/maps/tags` 서버 집계 API 추가. 필터 조건에 맞는 상위 태그를 반환.

## 9) UI 톤 통일
- 문제 -> 페이지네비게이션/검색/정렬/필터 스타일이 흐리거나 과도한 색감.
- 해결 -> 다크 그레이 톤으로 통일. 페이지네비게이션은 우측 정렬로 축소, 버튼은 다크 톤 유지.

## 10) 프리뷰 끄기 모드
- 문제 -> 프리뷰 사용 빈도가 낮고 리스트 집중 필요.
- 해결 -> 프리뷰 기본값 OFF. 끄면 리스트가 전체 너비로 확장.

## 11) 맵 합치기 UX (선택 모드)
- 문제 -> 다중 선택 후 합치기 흐름 필요.
- 해결 -> 상단 “선택” 버튼 추가, 선택 모드에서 체크박스 표시. 검색창 아래 액션바 제공.

## 12) 합치기 모달 UI
- 문제 -> 합치기 모달에서 순서 조정/루트 제목 입력 필요.
- 해결 -> 루트 제목 입력 + 드래그 앤 드롭 순서 변경 UI 제공. 합치기 버튼은 제목 미입력 시 비활성화.

## 13) 드래그 앤 드롭 라이브러리 선택
- 문제 -> 정렬을 드래그로 구현해야 함.
- 해결 -> 2026년 기준 유지보수/성능 측면에서 `@dnd-kit` 채택.

## 14) 합치기 순서 반전 이슈
- 문제 -> 합친 결과에서 순서가 역순으로 표시됨.
- 해결 -> 서버 병합 로직에서 배열 순서를 뒤집지 않고 원래 순서로 유지.

## 15) 검색 시 접힌 노드 에러
- 문제 -> `findEle`가 접힌 노드에서 throw 발생.
- 해결 -> `findEle` 호출을 안전하게 감싸고, 검색 시 경로를 자동 확장하여 노드를 펼치도록 처리.

### 비고
- 합치기 API: `/api/maps/merge`
- 태그 집계 API: `/api/maps/tags`
- 프리뷰 기본값: OFF
- 기본 기간: 최근 7일
