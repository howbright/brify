# 2026-03-10 맵 리스트 + 프리뷰 패널 개발로그

아래는 이 쓰레드의 진행을 `질문 -> 문제해결` 형식으로 정리한 개발로그입니다.

1. 질문: “이 UI/UX 의견만 먼저 말해줘.”
문제해결: 탐색 중심 구조(리스트+고정 프리뷰)가 목적에 맞다는 점을 확인하고, 스캔성/일관성/모바일 대응/상태 케이스 분리의 중요성을 강조.

2. 질문: “구현 명확하고 헷갈리는 거 없어? 자신 있어?”
문제해결: 구현 가능함을 확인하고, 바로 작업 진행 의사 확인.

3. 질문: “프리뷰가 안 뜨고 있어(로딩 중에서 멈춤).”
문제해결: 프리뷰 fetch effect 의존성/상태 전환을 안정화하여 로딩 상태가 정상 전환되도록 수정.

4. 질문: “프리뷰는 구조맵이어야 해. 루트+1레벨만 펼치고 하단에 2줄 구조화.”
문제해결: 텍스트 아웃라인 대신 실제 구조맵 미니 렌더로 변경하고, 루트+1레벨만 확장. 프리뷰 하단에 2줄 구조화 추가.

5. 질문: “프리뷰 세로 높이가 너무 작아.”
문제해결: 미니 프리뷰 최소 높이를 확대.

6. 질문: “프리뷰 닫기 기능 / 이동 모드 / 줌인아웃 필요.”
문제해결: 프리뷰 닫기/열기 UI, 줌 버튼 추가. 프리뷰 전용 pan 모드 및 줌 제어 핸들 구현.

7. 질문: “프리뷰에서 선택모드가 아니라 클릭 드래그로 이동되어야 함.”
문제해결: MindElixir의 mouseSelectionButton 동작을 반영해 좌클릭 드래그 팬이 되도록 설정 수정.

8. 질문: “줌 버튼을 우측 하단 플로팅으로.”
문제해결: 프리뷰 상단 버튼을 제거하고 미니맵 영역 우하단 플로팅 버튼으로 재배치.

9. 질문: “페이지 상단 제목이 헤더에 가려짐.”
문제해결: 상단 패딩을 늘려 제목 위치를 아래로 이동.

10. 질문: “리스트 스크롤 시 프리뷰가 안 보임.”
문제해결: 데스크톱에서 리스트만 스크롤되고 프리뷰는 고정(sticky)되도록 레이아웃 재구성.

11. 질문: “검색 입력이 흐릿함. 새 구조맵 버튼 제거.”
문제해결: 검색 입력 대비를 강화하고 페이지 내 생성 버튼 제거.

12. 질문: “검색 포커스 보더를 검정색으로.”
문제해결: 포커스 보더를 neutral 톤으로 변경하고 링 제거.

13. 질문: “리스트 가로 스크롤 제거 후 내용/보더가 잘림.”
문제해결: overflow-x 제거 및 텍스트/태그를 한 줄 말줄임 처리. 보더 잘림 원인 추적을 위해 min-w-0 / w-full / box-border 적용.

14. 질문: “아이템 오른쪽 보더 계속 잘림. 더 근본 원인 탐색.”
문제해결: 그리드 컨테이너와 리스트 섹션에 min-w-0/w-full 적용. 이후에도 문제가 지속되어 추가 구조 점검 진행.

15. 질문: “모바일에서 프리뷰가 리스트 하단에 나오는 UX가 부적절. 개선.”
문제해결: 모바일은 Bottom Sheet 프리뷰(선택 시 시트 열림, 배경 클릭 닫힘)로 변경.

16. 질문: “리스트 아이템 메뉴(… 삭제) 외부 클릭 시 닫히지 않음.”
문제해결: 외부 클릭/터치 감지로 메뉴 닫히도록 이벤트 리스너 추가.

## 주요 구현 산출물
- 맵 리스트 페이지: 리스트 + 고정 프리뷰(데스크톱), 모바일 bottom sheet
- MapListItem 컴포넌트: 선택 상태/말줄임/메뉴 외부 클릭 닫기
- MapPreviewPanel 컴포넌트: 닫기/열기, 줌 플로팅 버튼, 구조화 표시
- MapMiniPreview 컴포넌트: 실제 구조맵 미니 렌더, 루트+1레벨 확장
- ClientMindElixir 개선: 미니맵 표시 옵션, pan 모드 옵션
