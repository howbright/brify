# 2026-03-18 데모 페이지 흐름 정리 + 다크모드 개발로그

아래는 이 쓰레드의 진행을 `질문 -> 문제해결` 형식으로 정리한 개발로그입니다.

1. 질문: “demo 페이지가 마음에 들지 않는데, 입력타입 / 핵심 패턴 / 데모 방식 영역을 삭제해줘.”
문제해결: 히어로 상단의 `StatCard` 3개와 관련 컴포넌트를 제거해 첫 화면을 더 단순하게 정리.

2. 질문: “긴 유튜브 설명이 Brify에서 어떤 식으로 요약되고 구조화되는지... 문장 부분도 삭제해줘.”
문제해결: 히어로 설명 문단을 제거해 제목 중심 구조로 정리.

3. 질문: “이 페이지에서 작업 흐름이 보이지 않아. 썸네일, 스크립트 원문, 구조맵 생성 버튼, 구조맵 펼침, 노트/용어AI 기능 효용을 넣어줘.”
문제해결: 결과 소개형 페이지를 `입력 -> 생성 -> 활용` 흐름형 페이지로 재구성. 샘플 원문 스크립트를 데이터에 추가하고, Step 1~6 구조로 UI를 전면 개편.

4. 질문: “원문 스크립트가 없으면 달라고 요청해.”
문제해결: 저장된 전체 스크립트가 없음을 확인하고, 사용자가 제공한 스크립트를 `brifyDemoSample.ts`에 반영.

5. 질문: “페이지 컬러가 서비스 컬러와 맞지 않는다. 랜딩페이지와 globals.css를 참고해서 맞춰줘.”
문제해결: 하늘색/앰버/청록 중심 색을 줄이고, 서비스 전반의 블루 + 인디고 + 뉴트럴 톤으로 정리. CTA, 배경, 카드, 보더, 강조색까지 통일.

6. 질문: “구조맵 내용이 축소된 느낌이다. 내가 준 더 촘촘한 구조맵 데이터로 바꿔줘.”
문제해결: `brifyDemoSample.ts`의 `mindData`를 사용자가 제공한 상세 구조로 교체해 스크립트 밀도와 맞는 맵으로 변경.

7. 질문: “유튜브 썸네일 부분을 Step 1의 유튜브 영상 정보 확인으로 옮겨줘.”
문제해결: 히어로 오른쪽 썸네일 카드를 제거하고, Step 1 카드 내부에 썸네일/메타/태그를 통합.

8. 질문: “유튜브 설명이 아니라 유튜브 스크립트라고 해야 하지 않을까?”
문제해결: 히어로 제목을 `유튜브 스크립트를 넣으면, Brify가 이렇게 구조화합니다`로 수정.

9. 질문: “구조맵 영역이 너무 세로로 길다. 600px 정도로 줄여줘.”
문제해결: 맵 컨테이너 높이를 `600px`로 고정해 과도한 세로 확장을 방지.

10. 질문: “초기 글씨가 너무 작고, 클릭 드래그 이동도 안 된다.”
문제해결: `fitOnInit={false}`로 초기 축소를 제거하고 생성 직후 자동 확대. `dragButton={2}`, `panModeButton={2}`를 적용해 팬 모드 동작을 코드베이스 기준으로 바로잡음.

11. 질문: “Mind Elixir에서 제공하는 툴바를 보이게 해줘.”
문제해결: `ClientMindElixir`에 `showToolbar` prop을 추가해 툴바 표시를 props로 제어 가능하게 변경하고, 데모 페이지에서 활성화.

12. 질문: “데모페이지 구조맵 바로 위에 헤더를 만들고 이동/선택모드, 전체 펼치기, 전체 접기, 가운데 정렬, 확대, 축소 버튼을 넣어줘.”
문제해결: 구조맵 위에 데모 전용 조작 헤더를 추가하고, 맵 조작 액션을 한곳에 모은 컨트롤 바를 구현.

13. 질문: “선택 모드에서 선택 보더가 안 보인다.”
문제해결: `ClientMindElixir` 선택 상태에 대해 오버레이 보더와 그림자를 추가해 선택 노드가 시각적으로 분명하게 보이도록 보완.

14. 질문: “이동/선택 모드는 토글 버튼 하나보다 토글 바로 보여야 덜 헷갈린다.”
문제해결: 모드 전환을 단일 버튼이 아니라 세그먼트 토글 UI로 변경.

15. 질문: “선택 모드에서는 우클릭 드래그로 맵을 움직일 수 있다는 토스트가 필요하다. 그리고 img 경고도 고쳐줘.”
문제해결: 선택 모드 전환 시 안내 토스트를 추가하고, 썸네일을 `next/image`로 교체. 이후 `i.ytimg.com`를 `next.config.ts`에 허용해 SSR 에러까지 해결.

16. 질문: “원본 언어와 상관없이 원하는 언어로 결과를 받을 수 있다는 점을 알려주고, 한국어/영어를 선택하게 해줘.”
문제해결: Step 3에 `출력 언어 설정` UI를 추가하고, `summaryByLanguage`, `keyPointsByLanguage`, `mindDataByLanguage`를 도입해 실제 구조맵/요약이 한국어/영어로 전환되도록 구현.

17. 질문: “Step 3 설명 문장이 이해가 안 된다. 빼줘.”
문제해결: 불필요하게 추상적인 설명 문장을 제거해 생성 단계 설명을 더 직관적으로 정리.

18. 질문: “노드 선택 시 나오는 3개 버튼(노트/하이라이트/포커스)의 설명을 구조맵 하단에 간단하게 넣어줘.”
문제해결: 구조맵 하단에 선택 모드 안내 문구와 3개 액션 설명 카드를 추가.

19. 질문: “노트 기능은 노드 노트가 아니라 맵 전체 노트다.”
문제해결: Step 5를 `전체 노트 기능`으로 바꾸고, 설명도 전체 맵 메모 관점으로 수정.

20. 질문: “썸네일 적절한 곳에 유튜브 아이콘을 넣어줘.”
문제해결: Step 1 썸네일 왼쪽 상단에 유튜브 배지를 추가해 콘텐츠 출처를 더 직관적으로 표시.

21. 질문: “Step 2 원문 스크립트 박스 높이를 Step 1 하단과 맞게 조금씩 더 키워줘.”
문제해결: 스크립트 스크롤 영역 높이를 단계적으로 조정해 Step 1 카드 하단선과 비슷한 높이로 맞춤.

22. 질문: “Key Points는 실제 기능이 아니니 빼고 태그를 넣어줘.”
문제해결: 요약 옆 보조 카드에서 `Key Points`를 제거하고, 실제 기능을 반영한 `Tags` 영역으로 교체.

23. 질문: “상단 CTA 버튼(내 콘텐츠 무료 시작하기 / 원본 영상 보기)은 빼도 될까?”
문제해결: 데모 흐름 집중도를 높이기 위해 히어로 하단 CTA 2개를 제거하고, 하단 CTA만 유지.

24. 질문: “제목 옆 화살표를 누르면 ‘유튜브 스크립트’와 ‘어떤 긴 글도’ 두 버전이 swipe 되며 전환됐으면 좋겠다.”
문제해결: `framer-motion`을 이용해 제목 슬라이드 전환과 화살표 버튼을 구현하고, 화살표를 제목 바로 옆으로 재배치.

25. 질문: “맵 합치기 기능을 설명하고 싶은데, 문구와 작은 시각화도 넣고 싶다.”
문제해결: `More Than One Map` 카드를 추가하고, `맵 + 맵 = 연결된 맵` 형태의 미니 도식 시각화를 배치.

26. 질문: “태그 자동생성/분류도 설명하고 싶고, 비슷한 느낌의 시각화가 필요하다.”
문제해결: `Smart Tagging` 카드를 추가하고, 태그별 맵 군집을 단순한 아이콘 그룹으로 시각화. 이후 사용자 피드백에 맞춰 더 컴팩트하게 줄이고, 색 차이를 크게 벌리고, 연한 배경으로 그룹 단위를 다시 묶는 형태로 다듬음.

27. 질문: “다크모드 처리가 하나도 안 되어 있다.”
문제해결: 데모 페이지 전반에 `dark:` 변형을 추가해 배경, 카드, 보더, 버튼, 태그, 컨트롤 바, 시각화, CTA 섹션까지 다크 모드에 맞게 정리.

28. 질문: “포커스 버튼을 누른 뒤 선택 보더가 남아 있는 이상한 현상이 있다.”
문제해결: `ClientMindElixir`에서 포커스 진입/해제 시 선택 상태(`selectedNodeId`, `selectedRect`, `selectedNoteText`)를 정리하고, 포커스 모드 중에는 선택 오버레이를 렌더하지 않게 수정.

29. 질문: “하단 Next Step 영역이 다크모드에서 너무 안 튄다.”
문제해결: 다크 모드에서 CTA 섹션 배경 대비를 더 키우고, 블루/인디고 계열 글로우를 추가해 강조도를 높임.

30. 질문: “이 쓰레드 내용을 개발로그 md 파일로 만들어 docs 밑에 넣어줘.”
문제해결: 현재 쓰레드의 변경 흐름과 주요 수정 사항을 정리해 `/docs` 아래 오늘 날짜 기반 개발로그 파일로 생성.

## 주요 구현 산출물
- 데모 페이지 정보 구조 재설계: 입력 -> 원문 -> 생성 -> 구조맵 -> 활용 흐름
- 다국어 출력 데모: 한국어 / 영어 결과 전환
- 구조맵 전용 컨트롤 헤더: 이동/선택, 펼치기/접기, 중앙 정렬, 확대/축소
- 선택 오버레이와 포커스 모드 상태 정리
- 기능 소개 카드 추가: 전체 노트, 용어 AI 추출, 맵 합치기, 스마트 태깅
- 라이트/다크 모드 공통 대응

## 주요 수정 파일
- `app/[locale]/(public)/demo/page.tsx`
- `app/lib/demo/brifyDemoSample.ts`
- `components/ClientMindElixir.tsx`
- `next.config.ts`

## 확인 사항
- `npm run lint -- --file 'app/[locale]/(public)/demo/page.tsx'` 기준 경고/에러 없이 통과
- 외부 유튜브 썸네일은 `i.ytimg.com` 허용 설정으로 `next/image` SSR 오류 해결
