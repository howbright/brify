# 개발 로그 (2026-02-19)

## 문제 -> 해결방법

1. MindElixir 초기 렌더 타이밍에서 `me-root`가 없어 `offsetTop` 오류가 발생함.
해결: `me-root`가 생성될 때까지 최대 2초 대기 후 초기 스케일/센터링을 수행하고, 없으면 스킵하도록 가드 추가.

2. MindElixir에 전달되는 데이터 구조가 불확실해 원인 파악이 어려움.
해결: `ClientMindElixir` 초기화 직전에 `data`, `placeholderData`, `initialData` 로그를 추가.

3. 구조맵 상세 화면에서 한 문단 요약이 보이지 않음.
해결: `maps` 조회에 `summary` 필드 포함, `MapDraft`에 매핑하여 `LeftPanel`에서 렌더.

4. 요약 섹션 가독성이 떨어짐.
해결: 제목을 “한문단요약”으로 변경하고, 여백/패딩/줄간격을 조정해 읽기성 강화.

5. 처리중 상태가 지루하게 느껴짐.
해결: 기존 문장 교체는 유지하면서, 정제된 글자 수 기반 예상 시간으로 퍼센트 계산 및 표시.

6. 퍼센트 진행 상황의 시각화가 필요함.
해결: 퍼센트 뱃지 내부에 미니 프로그래스바를 함께 표시하도록 조정.

7. 서버 `source_char_count` 의존을 제거하고 클라이언트 정제 길이를 기준으로 처리하고 싶음.
해결: 크레딧 계산용 `normalizeForBilling` 결과 길이를 `sourceCharCount`로 저장해 퍼센트 계산에 사용하고, 서버 필드 의존 제거.
